**Agentix is the bridge between agents and the infrastructure around
them.** It runs agent CLIs and Python frameworks inside isolated
rollout containers, exposes them as typed Python functions, captures
LLM and tool activity as structured traces, and routes those traces to
evaluation harnesses, RL training buffers, observability, or custom
serving stacks.

Use the same bridge for Claude Code, Codex, Aider, mini-swe-agent,
OpenHands, your own agent, SWE-bench, internal benchmarks, and RL
post-training. Each agent, dataset, or tool is a normal Python
package; Agentix handles dispatch across the host-to-container
boundary.

## What It Bridges

| From | Through Agentix | To |
| --- | --- | --- |
| Agent CLIs and Python frameworks | Isolated namespace workers and typed remote dispatch | Evaluators, trainers, and orchestration code |
| Shell, file, and tool operations | Shared rollout container surface | Agents and benchmark harnesses |
| LLM calls and tool events | Structured trace capture and fan-out | Observability, reward, replay, and dataset pipelines |
| Rollout traces | [abridge](https://github.com/Agentiix/abridge) correlation and sink protocol | RL buffers such as slime / verl, or custom serving stacks |

## A Rollout Is Just Python

This SWE-bench Verified flow composes three integrations: prepare the
repo with `bash`, run Claude Code, then score the resulting patch with
the SWE-bench namespace.

```python
from datasets import load_dataset
from agentix import RuntimeClient, bash, claude_code, swebench

inst = dict(load_dataset("princeton-nlp/SWE-bench_Verified", split="test")[0])

async with RuntimeClient(sandbox.runtime_url) as c:
    await c.remote(
        bash.run,
        command=(
            f"git clone https://github.com/{inst['repo']}.git /testbed && "
            f"cd /testbed && git checkout {inst['base_commit']}"
        ),
    )
    cc = await c.remote(
        claude_code.run,
        instruction=inst["problem_statement"],
        workdir="/testbed",
        env={"ANTHROPIC_API_KEY": api_key},
    )
    diff = await c.remote(
        bash.run, command="cd /testbed && git add -A && git diff --cached",
    )
    score = await c.remote(swebench.score, instance=inst, patch=diff.stdout)
```

## Why Teams Use It

- **One bridge, many stacks.** Agents, tools, scorers, deployment
  backends, and trace sinks plug into the same runtime surface.
- **Typed dispatch.** `c.remote(fn, ...)` reads the function signature,
  so kwargs and return types flow into your editor and type checker.
- **Dependency isolation.** Each namespace runs with its own
  dependencies inside the rollout container, so incompatible agent
  stacks can ship together.
- **Training-ready traces.** Every LLM call and tool invocation can
  stream into [abridge](https://github.com/Agentiix/abridge), then into
  framework adapters for [slime](https://github.com/THUDM/slime),
  [verl](https://github.com/volcengine/verl), or a custom sink.
- **Composable evaluation.** Agent execution and benchmark scoring are
  separate namespaces, so you can swap agents, datasets, and backends
  independently.

## Supported Today

### Agents

- **Claude Code** — [recipe](https://github.com/Agentiix/agentix-cookbook/tree/main/claude-code)
- CLI binaries (Codex, Aider), Python frameworks
  ([mini-swe-agent](https://github.com/SWE-agent/mini-swe-agent),
  [swe-agent](https://github.com/SWE-agent/SWE-agent),
  [OpenHands](https://github.com/All-Hands-AI/OpenHands)),
  or your own — wrap with the
  [agent integration guide](/integrate-agent).

### Benchmarks

- **SWE-bench Verified** — [recipe](https://github.com/Agentiix/agentix-cookbook/tree/main/swebench),
  built on the official
  [`swebench`](https://github.com/swe-bench/SWE-bench) package.

### Primitives

- **bash** — shell execution inside the rollout container.
- **files** — upload, download, and edit files in the rollout container.

### Execution Backends

- `local` (Docker), `daytona`, `e2b` — built in.
- Third-party backends — `pip install agentix-deployment-<name>`.
