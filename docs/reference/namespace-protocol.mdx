A **namespace** is a normal Python distribution that contributes a class
to the `agentix.*` import namespace and declares one
`agentix.namespace` entry point. Callers invoke its methods via
`await c.remote(MyClass.method, x=1)` â€” the runtime resolves the
routing key from `method.__module__`, imports the class lazily, and
runs the real method body inside the sandbox.

## Package layout

A namespace is whatever `uv init --lib` produces, with one convention â€”
contribute to the `agentix.*` import namespace:

```text
agentix-myagent/
â”śâ”€â”€ pyproject.toml
â””â”€â”€ src/
    â””â”€â”€ agentix/              # PEP 420 namespace package â€” no __init__.py
        â””â”€â”€ myagent/
            â””â”€â”€ __init__.py   # class MyAgent(Namespace): ...
```

Why PEP 420: the framework's own `agentix/__init__.py` extends its
`__path__` via `pkgutil.extend_path`, so any wheel that installs files
at `<site-packages>/agentix/myagent/` becomes importable as
`from agentix.myagent import MyAgent`. Multiple namespace distributions
install peer entries under `agentix/` without colliding â€” the wheel's
own metadata, surfaced through `importlib.metadata`, is the only
discovery channel.

**Reserved framework subpackages** â€” namespace dists may not shadow:
`agentix.cli`, `agentix.deployment`, `agentix.dispatch`, `agentix.idents`,
`agentix.models`, `agentix.namespace`, `agentix.rollout`,
`agentix.runtime`, `agentix.trace`. Everything else
under `agentix.*` is fair game.

## The class IS the namespace

```python title="src/agentix/myagent/__init__.py"
from agentix.namespace import Namespace

class MyAgent(Namespace):
    """Optional docstring â€” surfaces in /namespaces output."""

    @staticmethod
    async def run(instruction: str) -> str:
        # real implementation â€” runs inside the sandbox
        return f"did: {instruction}"
```

- `class MyAgent(Namespace)` declares the namespace.
- Methods are `@staticmethod` â€” no `self`, no instance state.
- **Method bodies are the real implementation.** There is no stub vs
  impl split; no `_impl.py`, no `_register.py`, no
  `<Name>Impl` convention.
- Namespaces with heavy native dependencies should use **lazy imports
  inside method bodies** so caller-side `from agentix.myagent import MyAgent`
  stays cheap.

## Entry-point declaration

```toml title="pyproject.toml"
[project]
name = "agentix-myagent"
version = "0.1.0"
requires-python = ">=3.11"

[project.entry-points."agentix.namespace"]
myagent = "agentix.myagent:MyAgent"

[tool.hatch.build.targets.wheel]
packages = ["src/agentix"]
```

The key on the left (`myagent`) is the short name used in
`agentix build ... -o my-bundle:0.1.0` and shown by `agentix check`.
The value on the right (`agentix.myagent:MyAgent`) is what the runtime
imports.

## Sandbox-side runtime

Inside the sandbox, the runtime walks
`importlib.metadata.entry_points(group="agentix.namespace")` at startup
and records each `(name, module:class)` pair. **Nothing is imported
yet.** The first `POST /_remote` for a given namespace triggers
`ep.load()` under a per-namespace async lock, and the class becomes
live for the rest of the runtime's life. Subsequent dispatches reuse
the loaded class.

Import errors are cached on the registry entry â€” a broken namespace
fails calls to itself but never blocks boot or siblings. There is no
on-disk manifest the runtime inspects; the wheel's installed metadata
(visible through `importlib.metadata`) is the entire discovery
contract.

See [Architecture](/reference/architecture) for the full sandbox layout
(runtime + venv + optional `/nix/store`) and how `agentix build`
bundles multiple namespaces into a single deployable image.

## Wire

### Unary â€” `POST /_remote`

```text
POST /_remote
  { "package": "agentix.myagent",
    "method":  "run",
    "args":    [],
    "kwargs":  { "instruction": "fix the bug" } }
```

Success â€” always 200:

```json
{ "ok": true, "value": "did: fix the bug" }
```

Failure â€” also 200, with the exception class name in the payload:

```json
{ "ok": false, "error": { "type": "ValueError", "message": "...", "traceback": "..." } }
```

Only "package not found in registry" returns 404. W3C `traceparent`
and `tracestate` headers pass through.

### Streaming, bidirectional, log subscription â€” Socket.IO

`/socket.io/` carries every non-unary call. One persistent Socket.IO
connection per `RuntimeClient` multiplexes everything, demultiplexed by
a caller-generated `call_id`:

```text
stream            { call_id, package, method, args, kwargs }
stream:item       { call_id, value }
stream:end        { call_id }
stream:error      { call_id, error }

bidi:start        { call_id, package, method, args, kwargs }
bidi:in           { call_id, item }
bidi:end_in       { call_id }
bidi:out          { call_id, value }
bidi:end          { call_id }
bidi:error        { call_id, error }

logs:subscribe    { filter?: <logger-name prefix> }
log               { level, name, message, timestamp }
logs:unsubscribe  {}
```

`RuntimeClient.remote` picks the wire automatically by inspecting the
method's return annotation: plain return â†’ unary HTTP;
`AsyncIterator[T]` â†’ server-streaming Socket.IO; `AsyncIterator[U]`
parameter + `AsyncIterator[V]` return â†’ bidi.

## Caller side

```python
from agentix import RuntimeClient
from agentix.myagent import MyAgent

async with RuntimeClient(sandbox.runtime_url) as c:
    result = await c.remote(MyAgent.run, instruction="â€¦")
    # result: str â€” pyright infers from MyAgent.run's return type
```

`RuntimeClient.remote(fn, *args, **kwargs)`:

- Reads `fn.__module__` â†’ routing key (the entry-point value's left side).
- Reads `fn.__name__` â†’ method.
- Serialises args/kwargs via pydantic `TypeAdapter` driven by
  `inspect.signature(fn)`.
- Decodes the response into `fn`'s return type.

No magic registration, no decorators on the class. The function
reference and its signature carry everything the wire needs.

## Streaming methods

A method annotated `-> AsyncIterator[T]` (or `AsyncGenerator[T, ...]`)
opts into the streaming wire.

```python title="src/agentix/chatbot/__init__.py"
from collections.abc import AsyncIterator
from agentix.namespace import Namespace

class Chatbot(Namespace):
    @staticmethod
    async def chat(prompt: str) -> AsyncIterator[str]:
        proc = await asyncio.create_subprocess_exec("claude", "-p", prompt, ...)
        async for line in proc.stdout:
            yield line.decode()
```

Caller iterates directly â€” **no `await` on the call site**:

```python
async for token in c.remote(Chatbot.chat, prompt="â€¦"):
    print(token, end="")
```

Items flow over the Socket.IO `stream:item` event. The stream closes
with `stream:end`; impl exceptions arrive as `stream:error` and the
client raises `RemoteCallError`. Already-yielded items are observed
before the raise.

## Bidirectional methods

A method whose **first parameter** is `AsyncIterator[U]` and whose
**return type** is `AsyncIterator[V]` opts into bidi:

```python
async def echo(messages: AsyncIterator[str]) -> AsyncIterator[str]:
    async for msg in messages:
        yield f"echo: {msg}"
```

The caller passes an async iterable of inputs and consumes another
async iterable of outputs over the same `call_id`. Inputs flow as
`bidi:in` events; outputs as `bidi:out`.

## Runtime built-ins

Independent of any namespace, the runtime exposes these endpoints â€”
useful when you need raw sandbox I/O without going through a namespace.

| Endpoint              | Purpose                                                                                                                                              |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `GET /health`         | Liveness.                                                                                                                                            |
| `POST /exec`          | Run a shell command in the sandbox. Body `{command, cwd?, env?, timeout?}`. SSE when `Accept: text/event-stream`; else JSON `{exit_code, stdout, stderr}`. |
| `POST /upload`        | Multipart upload into `AGENTIX_UPLOAD_ROOT` (default `/workspace`).                                                                                  |
| `GET /download?path=` | Stream a file back.                                                                                                                                  |
| `GET /namespaces`     | List loaded namespaces.                                                                                                                              |

`RuntimeClient.run / upload / download / namespaces` are typed Python
helpers over these. Directory listing and file inspection go through
`/exec` (`ls -la`, `find`, `stat`).

### `/exec` env and PATH policy

Subprocesses run with a scrubbed env:

- **Stripped:** `LD_LIBRARY_PATH`, `LD_PRELOAD`, `PYTHONPATH`,
  `PYTHONHOME`, `LOCALE_ARCHIVE`, `FONTCONFIG_*`, `SSL_CERT_FILE`,
  anything prefixed `NIX_`. The runtime is a Nix-built binary; leaking
  those into a task-image subprocess causes glibc ABI mismatches.
- **Default PATH:** the task image's `/usr/local/bin:/usr/bin:/bin`.
  Task-image tools take precedence over namespace-bundled tools of the
  same name.
- **Namespace native tools:** ship at content-addressed
  `/nix/store/<hash>/bin/<tool>` paths and are invoked by their absolute
  path from Python impls. Content-addressed paths are stable across the
  bundle's lifetime.

## Minimal recipe

```text
my-namespace/
â”śâ”€â”€ pyproject.toml          # [project.entry-points."agentix.namespace"]
â””â”€â”€ src/agentix/<name>/
    â””â”€â”€ __init__.py         # class MyClass(Namespace) with @staticmethod methods
```

That's it. `pip install ./my-namespace` makes it discoverable; `pytest`,
`pyright`, `ruff`, `uv build` all work against the source directory
without further configuration.

To ship a deployable image, drop in the shared
[`primitives/_template/Dockerfile`](https://github.com/Agentiix/Agentix/tree/master/primitives/_template)
and
[`primitives/_template/default.nix`](https://github.com/Agentiix/Agentix/tree/master/primitives/_template),
then `agentix build ./my-namespace`. `primitives/bash` and
`primitives/files` are working in-tree examples.
