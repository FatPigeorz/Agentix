The Agentix CLI has one verb:

- `agentix build` turns a Python project into a deploy-ready runtime
  image.

```bash
agentix --help
agentix build --help
```

## `agentix build`

Package one project root into a bundle image.

```bash
agentix build
agentix build path/to/project
agentix build . -o my-rollout:0.1.0
agentix build . --dry-run
agentix build . --runtime-image agentix/runtime:0.2.0
```

The project root must contain `pyproject.toml`. The default image tag is
derived from `[project].name` and `[project].version`; `-o`, `--output`,
or `--tag` overrides it.

| Option | Meaning |
| --- | --- |
| `path` | Project root, default `.` |
| `-o`, `--output`, `--tag` | Output image tag, must include `:<version>` |
| `--runtime-image` | Base runtime image to extend |
| `--dry-run` | Stage the Dockerfile and build context without running Docker |

### Dependency Model

The CLI never enumerates integrations. It installs the project into the
runtime venv, and pip resolves everything declared by the project.

```toml title="pyproject.toml"
[project]
name = "my-rollout"
version = "0.1.0"
dependencies = [
    "agentixx>=0.1.0",
    "agentix-runtime-basic>=0.1.0",
    "agentix-claude-code>=0.1.0",
    "agentix-swebench>=0.1.0",
]
```

That project builds one image containing the framework, sandbox
primitives, the agent wrapper, the scorer wrapper, transitive deps, and
the project itself.

### Dry Run

```bash
agentix build . --dry-run
```

Dry run stages the generated Dockerfile and copied project tree under
`./build/<tag>/`, prints what would be built, and exits without invoking
Docker.

## Configuration

Most configuration is environment-driven so CLI and code paths stay the
same.

| Variable | Used by | Purpose |
| --- | --- | --- |
| `AGENTIX_BIND_PORT` | runtime server | Sandbox-side bind port, default `8000` |
| `AGENTIX_UPLOAD_ROOT` | file primitive packages | Sandbox-side root for file operations |
| `DAYTONA_API_KEY` | `daytona` backend | API authentication |
| `E2B_API_KEY` / `E2B_TEMPLATE_ID` | `e2b` backend | API authentication and template selection |

Third-party packages that need their own top-level workflow should ship a
separate console script such as `agentix-yourcmd` instead of expanding the
core CLI.
