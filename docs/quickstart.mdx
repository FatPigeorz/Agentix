## Prerequisites

- Python 3.11 or newer
- Docker (for the `local` deployment)

This quickstart proves the smallest Agentix bridge: host-side Python
calls a sandboxed namespace as if it were a local typed function.

## Install

<Tabs>
  <Tab title="pip">
    ```bash
    pip install agentix agentix-runtime-basic agentix-deployment-docker
    ```
  </Tab>
  <Tab title="uv">
    ```bash
    uv add agentix agentix-runtime-basic agentix-deployment-docker
    ```
  </Tab>
  <Tab title="From source">
    ```bash
    git clone https://github.com/Agentiix/Agentix
    cd Agentix && pip install -e '.[dev]'
    pip install -e ../Agentix-Runtime-Basic
    ```
  </Tab>
</Tabs>

## Verify

```bash
agentix check
```

```text
OK   bash    agentix.bash    agentix-runtime-basic@0.1.0
OK   files   agentix.files   agentix-runtime-basic@0.1.0
```

## Call a namespace

```python title="hello_sandbox.py"
import asyncio
from agentix import RuntimeClient, SandboxConfig, bash
from agentix.deployment.base import session
from agentix.deployment.docker import DockerDeployment

async def main():
    deployment = DockerDeployment()
    config = SandboxConfig(image="my-bash:0.1.0")   # produced by `agentix build path/to/my-project`
    async with session(deployment, config) as sandbox:
        async with RuntimeClient(sandbox.runtime_url) as c:
            r = await c.remote(bash.run, command="echo hi")
            print(r.stdout)   # → "hi\n"

asyncio.run(main())
```

```bash
python hello_sandbox.py
# → hi
```

First run pulls the bundle image (~30 s). Subsequent sandboxes start
in ~100 ms.

## Write your own namespace

<Tabs>
  <Tab title="Source">
    ```python title="src/agentix/myagent/__init__.py"
    async def run(instruction: str) -> str:
        return f"did: {instruction}"
    ```
  </Tab>
  <Tab title="pyproject.toml">
    ```toml
    [project]
    name = "agentix-myagent"
    version = "0.1.0"

    [project.entry-points."agentix.namespace"]
    myagent = "agentix.myagent"

    [tool.hatch.build.targets.wheel]
    packages = ["src/agentix"]
    ```
  </Tab>
  <Tab title="Layout">
    ```text
    my-namespace/
    ├── pyproject.toml
    └── src/
        └── agentix/                  # PEP 420 namespace package, no __init__.py
            └── myagent/
                └── __init__.py
    ```
  </Tab>
</Tabs>

```bash
agentix build ./my-namespace                       # → myagent:0.1.0
agentix deploy local --image myagent:0.1.0
```

Bundling several plugins? Declare them in a project's `pyproject.toml`
under `[project].dependencies` and `agentix build` that project — pip
resolves transitively.

`pip install agentix-myagent` is everything your users need. The
framework discovers the entry point, and
`from agentix.myagent import run` resolves natively in their code.
