---
title: Quickstart
description: Build a tiny Python bundle, deploy it in Docker, and call it with RuntimeClient.remote.
---

This quickstart builds the smallest useful Agentix flow:

1. Create a normal Python package with one async function.
2. Build it into a sandbox image with `agentix build`.
3. Start a local Docker sandbox.
4. Call the function from host-side Python with `RuntimeClient.remote`.

## Prerequisites

- Python 3.11 or newer
- Docker
- An Agentix runtime base image such as `agentix/runtime:0.1.0` available
  locally or in your registry

## Install

Install the host-side framework and the Docker deployment backend:

<Tabs>
  <Tab title="pip">
    ```bash
    pip install agentixx agentix-deployment-docker
    ```
  </Tab>
  <Tab title="uv">
    ```bash
    uv add agentixx agentix-deployment-docker
    ```
  </Tab>
  <Tab title="From source">
    ```bash
    git clone https://github.com/Agentiix/Agentix
    cd Agentix
    pip install -e '.[dev]'
    pip install -e ../Agentix-Deployment-Docker
    ```
  </Tab>
</Tabs>

If your remote function imports sandbox primitives such as `agentix.bash`,
also install and declare `agentix-runtime-basic`.

## Create a Remote Target

Create a package that exposes one function. There is no base class,
registration hook, or framework marker.

<Tabs>
  <Tab title="src/hello_agentix/__init__.py">
    ```python
    async def run(input: str) -> str:
        return f"sandbox saw: {input}"
    ```
  </Tab>
  <Tab title="pyproject.toml">
    ```toml
    [project]
    name = "hello-agentix"
    version = "0.1.0"
    requires-python = ">=3.11"
    dependencies = ["agentixx>=0.1.0"]

    [build-system]
    requires = ["hatchling"]
    build-backend = "hatchling.build"

    [tool.hatch.build.targets.wheel]
    packages = ["src/hello_agentix"]
    ```
  </Tab>
  <Tab title="Layout">
    ```text
    hello-agentix/
    ├── pyproject.toml
    └── src/
        └── hello_agentix/
            └── __init__.py
    ```
  </Tab>
</Tabs>

Install the package locally too. The caller imports the same function
object it will ask the sandbox to run.

```bash
pip install -e ./hello-agentix
```

## Build the Bundle

`agentix build` installs the project and its dependencies into one image.

```bash
agentix build ./hello-agentix -o hello-agentix:0.1.0
```

If your project has a `default.nix`, Agentix adds a Nix builder stage and
links the derivation's binaries into the final image under
`/nix/runtime/bin`.

## Call It

Deploy the image with the Docker backend, then call the remote function.

```python title="call_hello.py"
import asyncio

from agentix import RuntimeClient
from agentix.deployment.base import SandboxConfig, session
from agentix.deployment.docker import DockerDeployment
from hello_agentix import run


async def main() -> None:
    deployment = DockerDeployment()
    config = SandboxConfig(image="hello-agentix:0.1.0")

    async with session(deployment, config) as sandbox:
        async with RuntimeClient(sandbox.runtime_url) as client:
            result = await client.remote(run, input="hello")
            print(result)


asyncio.run(main())
```

```bash
python call_hello.py
# sandbox saw: hello
```

The client sends the target `hello_agentix::run` to the runtime worker.
The worker imports `hello_agentix`, validates the kwargs against
`run`'s signature, calls it, and returns the decoded `str`.
