## Prerequisites

- Python 3.11+
- [Docker](https://docs.docker.com/get-docker/) — the only host
  requirement. Nix runs inside the builder stage of
  [`primitives/_template/Dockerfile`](https://github.com/Agentiix/Agentix/tree/master/primitives/_template).

## Setup

```bash
git clone https://github.com/Agentiix/Agentix.git
cd Agentix
pip install -e '.[dev]'                        # fastapi, uvicorn, httpx, pytest, ruff
pip install -e primitives/bash primitives/files
```

## Daily workflow

### Run the runtime server locally (no sandbox)

```bash
agentix-server                 # http://localhost:8000
agentix-server --port 9000
```

Without any namespaces installed the server exposes its built-ins only
(`/health`, `/exec`, `/upload`, `/download`). Useful for iterating on
the runtime without the Docker round-trip; `pip install -e primitives/bash`
makes `agentix.bash:Bash` discoverable on the next start.

### Build namespace images

The runtime image and the in-tree primitives share one Dockerfile:

```bash
agentix build primitives/bash                          # → agentix/bash:0.1.0
agentix build primitives/files                         # → agentix/files:0.1.0
docker build -t agentix/runtime:dev -f primitives/_template/Dockerfile .
```

### Smoke-test end-to-end in Docker

```python
import asyncio
from agentix import RuntimeClient, SandboxConfig
from agentix.deployment.base import session
from agentix.deployment.docker import DockerDeployment
from agentix.bash import Bash

async def main():
    deployment = DockerDeployment()
    config = SandboxConfig(
        image="ubuntu:24.04",
        runtime="agentix/runtime:dev",
        namespaces=["agentix/bash:0.1.0"],
    )
    async with session(deployment, config) as sb:
        async with RuntimeClient(sb.runtime_url) as c:
            print(await c.run("uname -a"))
            result = await c.remote(Bash.run, command="echo hi")
            print(result.stdout)

asyncio.run(main())
```

### Lint & test

```bash
ruff check agentix/ tests/ primitives/
pytest                      # unit tests
pytest -x                   # stop on first failure
```

## Adding a new namespace

A namespace is a normal Python project (whatever `uv init --lib`
produces) with one entry-point declaration. The fastest path is to
copy the template:

```bash
cp -r primitives/_template my-namespace
# edit my-namespace/pyproject.toml (name, entry point)
# edit my-namespace/src/agentix/<name>/__init__.py (your class)
agentix build my-namespace
```

Use it:

```python
SandboxConfig(
    image="ubuntu:24.04",
    runtime="agentix/runtime:dev",
    namespaces=["my-namespace:0.1.0"],
)
```

See [Namespace protocol](/reference/namespace-protocol) for the full
contract, and [`primitives/bash`](https://github.com/Agentiix/Agentix/tree/master/primitives/bash)
for a working class-based example.
